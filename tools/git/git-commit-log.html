注意

要避免强推，最好是创建和推(push)一个新的提交(commit)，而不是强推一个修正后的提交。后者会使那些与该分支或该分支的子分支工作的开发者，在源历史中产生冲突。 最好是创建和推(push)一个新的提交(commit)
